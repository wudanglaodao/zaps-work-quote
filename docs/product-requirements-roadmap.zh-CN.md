# zaps.work 产品需求与开发路线图

> 版本：v1.0<br>
> 更新时间：2026-07-14<br>
> 状态：规划中

## 1. 产品定位

zaps.work 是面向小型服务商、独立工作者与小型制造商的免费、多语言成本计算与客户报价工具库。

用户不需要注册：在浏览器内输入实际成本、人工、损耗、利润与税率，立即得到可解释的建议报价，并导出可发送给客户的 PDF 或 CSV。

### 不做什么

- 暂不做 CRM、客户跟进、在线签约、支付、发票或完整单据工作流。
- 暂不以“万能报价单生成器”为下一步；优先做规则明确、能独立获得搜索流量的垂直报价工具。
- 不提供虚假的行业均价或承诺最终成交价；所有结果必须明确由用户自己的费率与假设得出。

## 2. 已上线工具分类（8 个）

产品已经不是单一的“报价单工具”，而是三类不同计价模型的工具库。分类应以用户如何报价为主，而不是只按行业名称排列。

| 工具族 | 已上线工具 | 主要用户 | 共同计价模型 | 可连续拓展方向 |
| --- | --- | --- | --- | --- |
| **实物生产与制造**（3） | 3D 打印、激光切割、CNC 加工 | Maker、小型加工厂、工程服务商 | 材料 + 机器时间 + 人工 + 损耗 + 批量摊销 + 利润 | 钣金折弯、焊接、UV 打印、木工 |
| **现场服务与房屋维护**（4） | 房屋清洁、房屋油漆、草坪服务、高压清洗 | 本地服务个体户、小型班组 | 面积/数量 + 作业复杂度 + 班组工时 + 交通/设备 + 最低收费 + 利润 | 窗户清洁、天沟清洁、围栏、地板/瓷砖 |
| **知识与创意服务**（1） | 自由职业项目报价 | 设计师、开发者、顾问、接单型自由职业者 | 工作范围 + 工时费率 + 修改 + 固定成本 + 加急 + 利润 | 摄影、视频、网站项目、翻译/本地化 |

### 2.1 制造加工工具

| 工具 | 用户要解决的问题 | 当前差异化字段 | 与其他工具的关系 |
| --- | --- | --- | --- |
| 3D 打印成本计算器 | 单件或多件打印不能只看耗材价格 | 耗材、机器时间、电力、失败风险、后处理 | 制造入口工具，适合 Maker 与原型用户 |
| 激光切割成本计算器 | 板材加工容易漏算 CAD/CAM、切割与损耗 | 板材、切割数据、设置、废料、利润 | 可衔接钣金折弯、焊接 |
| CNC 加工成本计算器 | 批量加工需要摊销编程、设置和机时 | 材料体积、编程、设置、循环时间、检验、刀具 | 高客单制造工具，可衔接复杂加工场景 |

### 2.2 现场服务与房屋维护工具

| 子类 | 工具 | 用户要解决的问题 | 当前差异化字段 |
| --- | --- | --- | --- |
| 室内维护 | 房屋清洁报价 | 服务频率、房间与附加项如何标准化报价 | 房间、面积、频率、附加服务、清洁工时 |
| 室内维护 | 房屋油漆报价 | 面积、涂层与墙面状态如何转化为报价 | 覆盖率、涂层、表面状况、材料、人工 |
| 户外维护 | 草坪服务报价 | 草坪大小和草况如何转化为班组工时 | 面积、草况、生产率、设备、交通 |
| 户外维护 | 高压清洗报价 | 不同污渍/面积和设备成本如何报价 | 车道面积、污渍状况、班组时间、运营成本 |

这四个工具已形成“本地服务报价”系列。下一步应优先补充相邻但不重复的场景，避免再做泛化的清洁工具。

### 2.3 知识与创意服务工具

| 工具 | 用户要解决的问题 | 当前差异化字段 | 可延展方向 |
| --- | --- | --- | --- |
| 自由职业项目报价 | 防止按工时报价时漏掉修改、固定成本和加急 | 工作时数、额外修改、修改时数、急件加价、固定成本 | 摄影、视频、设计、开发、咨询 |

### 2.4 横向共用能力

所有工具必须共用并持续保持以下能力：

- 多货币与多语言界面；使用正确的本地单位。
- 直接成本、目标利润率、最低收费、税额和总价。
- 客户、公司、报价编号与有效期；信息只保留在浏览器。
- 客户可核对的 PDF/CSV：报价行之和等于未税小计，税额只出现一次。
- 移动端实时报价条、异常输入保护、可复核的成本拆解。

### 2.5 产品内归档与分类（下一步优先做）

内部分类必须变成用户看得见、能导航的工具归档，而不只是开发文档。

#### 信息架构

```text
所有工具
├─ 制造与加工（3）
│  ├─ 3D 打印成本
│  ├─ 激光切割成本
│  └─ CNC 加工成本
├─ 本地服务与房屋维护（4）
│  ├─ 房屋清洁报价
│  ├─ 房屋油漆报价
│  ├─ 草坪服务报价
│  └─ 高压清洗报价
└─ 自由职业与专业服务（1）
   └─ 自由职业项目报价
```

#### 首次上线范围

1. **所有工具页**：从单一列表升级为分类工具库；默认展示三个分类和每类工具数量。
2. **首页**：继续保留“1 个主推 + 4 个精选”，但“所有工具”入口进入分类工具库，而不是平铺清单。
3. **工具详情页**：显示所属分类，并在页尾推荐 2–3 个同类或相邻工具。
4. **多语言**：分类名称、描述、工具数量和推荐区必须随语言切换。
5. **SEO**：不新建空泛、低价值的分类 URL；先在 `/tools` 完成可索引的分类锚点与内部链接，等每类至少 4 个工具后再考虑独立分类落地页。

#### 验收标准

- 用户从首页两次点击内能到达任意已上线工具。
- 用户能在不理解工具名称时，通过“制造 / 本地服务 / 自由职业”找到正确入口。
- 不改变任何既有工具 URL、canonical、Sitemap 或已收录页面。
- 移动端能快速展开、浏览和进入三个分类。

## 3. 目标用户与核心任务

| 用户 | 真实任务 | 成功标准 |
| --- | --- | --- |
| 本地服务个体户/小团队 | 现场快速算出不亏钱的服务价格 | 2 分钟内完成输入并能发出报价 |
| 小型制造商/Maker | 把材料、机器、人工与批量摊销算清 | 报价能解释成本与目标利润 |
| 自由职业者 | 管住工时、修改、加急与固定成本 | 不再只按工时漏算隐性成本 |
| 潜在客户 | 获得可沟通的价格范围 | 清楚知道报价包含的服务与税费 |

优先把同一套可解释模型本地化，而不是先做完全独立的日本产品。

- 空调清洗报价（エアコンクリーニング）
- 搬家服务报价（引越し）
- 草刈り报价
- 不用品回收报价
- ハウスクリーニング与窗户清洁长尾页

日语版本必须使用日元、平方米/日语服务单位、消费税输入与自然日语的报价单字段。

## 5. 开发优先级与节奏

### 阶段 -1：工具库归档与分类（先做，约 2–3 天）

1. 将 `/tools` 改为三类工具归档页：制造与加工、本地服务与房屋维护、自由职业与专业服务。
2. 首页的“所有工具”统一进入分类工具库；精选区只承担发现与推荐，不承担完整目录。
3. 每个工具页增加所属分类和相邻工具推荐。
4. 保持既有工具 URL 和 Sitemap 不变，只增强内部链接、页面语义与多语言文案。

### 阶段 0：可信度与数据基础（与归档并行）

1. 发布自由职业、CNC、草坪工具的报价可信度修复。
2. 为这三个工具补齐匿名的 `calculator_used`、`pdf_exported`、`csv_exported` 事件。
3. 每周查看：工具访问、完成输入、PDF/CSV 导出率、语言与货币分布。

### 阶段 1：窗户清洁报价计算器（已开发，待发布）

选择理由：规则具体、用户能自行输入、与现有本地服务模块复用度高，同时不会和清洁/高压清洗工具重复同一搜索意图。

#### 用户故事

- 作为窗户清洁服务商，我想按窗户类型、楼层和附加服务计算报价，避免漏掉高空与细节清洁时间。
- 作为服务商，我想把公司、客户、报价编号和有效期写入 PDF，能在现场直接发给客户。
- 作为客户，我想从报价单看懂服务范围、未税小计、税额和总价。

#### 输入字段

| 分组 | 字段 |
| --- | --- |
| 服务范围 | 标准窗数量、大玻璃/推拉门数量、天窗数量、内侧/外侧/双面 |
| 作业复杂度 | 楼层或高空作业、窗户脏污程度、是否需要梯子 |
| 附加项 | 纱窗、深度清洁轨道、窗框、硬水渍/顽固污渍 |
| 班组与成本 | 每类服务单价或工时、班组人数、人工费率、交通、设备成本 |
| 定价 | 间接成本、目标利润率、最低收费、税率 |
| 报价信息 | 公司、客户、联系方式、报价编号、有效天数 |

#### 计算规则

1. 按不同窗型和清洁面数计算基础工时或基础服务费。
2. 对楼层、梯子和脏污情况施加明确的复杂度系数。
3. 加入纱窗、轨道、窗框等可见附加服务。
4. 直接成本 = 人工 + 交通 + 设备 + 可选耗材。
5. 未税报价 = `max(直接成本 / (1 - 目标利润率), 最低收费)`。
6. 税额只在报价底部单独计算；客户报价行之和必须等于未税小计。

#### 输出与验收

- 实时报价：直接成本、利润、利润率、未税小计、税额、总价。
- 客户报价：服务行、服务范围说明、客户信息、有效期、税额与总价。
- CSV：成本项目、定价调整、税额与总价；不得把税额重复写入项目行。
- 支持货币、英制/公制切换、移动端报价条与异常输入保护。

### 阶段 2：内容与验证（1 周）

为窗户清洁工具同步交付：

1. 英文操作指南 `How to Price Window Cleaning Jobs`。
2. 3–5 个 FAQ/长尾主题：按窗收费、二层窗、纱窗、内外双面、轨道深度清洁。
3. 更新工具 Sitemap、指南 Sitemap、总 Sitemap 与 `llms.txt`。
4. 检查所有语言路由、canonical、JSON-LD、移动端与 PDF。

### 阶段 3：双线扩展（后续 4–6 周）

| 顺序 | 工具 | 目的 |
| --- | --- | --- |
| 2 | 摄影项目报价 | 补足创意人群，验证自由职业线的延展性 |
| 3 | 钣金折弯成本计算器 | 加深制造工具线，与激光/CNC 互相导流 |
| 4 | 天沟清洁报价 | 复用高空作业与本地服务能力，形成服务系列 |

只有当窗户清洁或摄影工具达到可观的使用/导出信号后，才继续扩展对应系列；否则回到表现更好的制造或本地服务线。

## 6. 每个新工具的标准交付清单

### 产品可信度

- [ ] 每个字段都确实影响结果，或在界面中明确说明用途。
- [ ] 无效、空白、极大和负数输入不会导致崩溃或 NaN。
- [ ] 货币切换会转换金额字段；单位切换会转换数值而非只换标签。
- [ ] 报价行之和 = 未税小计；税只出现一次；总价可复核。
- [ ] PDF、CSV、预览与移动端显示使用同一套计算结果。
- [ ] 客户信息仅保留在浏览器内，不进入分析数据。

### SEO 与内容

- [ ] 工具元数据、结构化数据、canonical 与多语言 hreflang。
- [ ] 工具 Sitemap、指南 Sitemap、总 Sitemap、`llms.txt` 已更新。
- [ ] 至少一篇操作指南和一组 FAQ/长尾问答。
- [ ] 与同场景工具互相链接，不制造孤岛页面。

### 发布与观察

- [ ] 单元测试、类型检查、lint、生产构建通过。
- [ ] 移动端、PDF、CSV 与至少一个非英语语言人工验收。
- [ ] 记录匿名使用与导出事件。
- [ ] 发布后第 7 天、第 28 天复盘搜索展示、点击、工具使用与导出率。

## 7. 决策门槛

在没有稳定使用信号前，不扩张到账号体系或单据工作流。优先用以下指标决定下一轮：

| 信号 | 下一步 |
| --- | --- |
| 工具使用多、导出率高 | 扩展同一场景族的相邻工具 |
| 搜索展示多、点击低 | 优化标题、描述、FAQ 与首屏匹配度 |
| 使用多、导出少 | 审计报价可读性、客户信息和移动端流程 |
| 某语言持续增长 | 为该语言补本地单位、术语与专门长尾页 |
| 用户反复需要多行自定义项目 | 再评估轻量多行报价能力，而非直接做 CRM |

# zaps.work Prompt Pack

Use these prompts to refine the zaps.work homepage, calculator pages, and design system.

## 1. Homepage Direction

```text
Design a clean homepage for zaps.work, a practical cost calculator and quote generator platform.

The page should feel like a serious work tool for makers, freelancers, and small businesses, not an AI SaaS landing page.

Use a light-mode interface first. The first viewport should combine:
- a concise headline
- a tool search input
- category filters
- available and coming-soon tool cards
- a realistic quote calculator preview on the right

Visual style:
- Vercel-level clarity
- fintech dashboard polish
- restrained green accent
- precise spacing
- neutral surfaces
- subtle borders
- minimal shadows

Avoid:
- AI chatbot UI
- glowing gradients
- abstract blobs
- oversized marketing hero
- stock illustrations
- purple SaaS palette
- vague "AI-powered" copy
```

## 2. Homepage Polish

```text
Refine this homepage so it feels more premium, practical, and trustworthy.

Keep the structure tool-first:
- search tools
- tool categories
- popular tool cards
- quote workspace preview

Improve:
- spacing rhythm
- typography hierarchy
- button sizing
- disabled coming-soon states
- calculator preview density
- quote document preview
- mobile responsiveness

The result should feel like a useful workbench, not a marketing page.
Do not add decorative gradients, orbs, chat bubbles, or AI assistant patterns.
```

## 3. Less AI, More Utility

```text
Remove the "AI product" feeling from this design.

Make it feel like:
- a pricing calculator directory
- a quoting workspace
- a professional estimate tool
- a small business utility

Use direct, concrete language:
- "Calculate job cost"
- "Generate quote"
- "Export PDF"
- "Cost breakdown"
- "Profit and margin"

Avoid:
- "Automate your workflow"
- "AI-powered"
- "Generate anything"
- "Chat with your data"
- abstract productivity language

Keep the interface calm, dense, and functional.
```

## 4. Tool Card System

```text
Design a tool card system for zaps.work.

Each available tool card should show:
- tool name
- one-line use case
- output type
- status badge

Available tools should feel clickable.
Coming-soon tools should use disabled styling:
- muted background
- muted text
- subtle border
- small "Soon" badge
- no hover lift
- not clickable

The cards should be easy to scan and should not feel like generic SaaS feature cards.
```

## 5. Calculator Preview

```text
Design a compact calculator preview for the zaps.work homepage.

Use the 3D Print Cost Calculator as the example.

The preview should include:
- title
- tabs: Input, Breakdown, Quote
- job detail inputs
- suggested quote
- total cost
- profit
- margin
- cost breakdown bars
- quote PDF preview

The UI should look realistic and usable, not like a decorative mockup.
Use white surfaces, subtle gray borders, black text, and green only for profit/accent states.
```

## 6. Dark Mode Version

```text
Create a dark mode version of the zaps.work homepage.

Keep quote documents white even in dark mode.

Dark mode should use:
- near-black / deep green-black page background
- dark neutral panels
- readable white text
- subtle borders
- green accent for profit and active states

Avoid:
- purple-blue gradients
- neon cyber style
- glowing cards
- heavy glassmorphism

The page should still feel like a professional cost and quote tool.
```

## 7. Copywriting Pass

```text
Rewrite the homepage copy for zaps.work.

Positioning:
zaps.work is a practical platform for cost calculators and quote generators.

Audience:
makers, small shops, freelancers, and local service businesses.

Tone:
plain, useful, confident, not hype-driven.

Avoid AI buzzwords.
Avoid generic SaaS copy.
Avoid abstract productivity claims.

Create:
- headline
- subheadline
- search placeholder
- primary CTA
- tool card descriptions
- section titles for Cost Layer and Quote Layer
```

## 8. Visual Critique Prompt

```text
Review this zaps.work homepage design as a senior product designer.

Focus only on practical UI quality:
- hierarchy
- spacing
- contrast
- scanability
- disabled states
- density
- mobile behavior
- whether it feels too AI/SaaS/marketing-heavy

Do not suggest broad rebrands.
Do not add new product concepts.
Give specific, actionable improvements.
```

## 9. Mobile Homepage

```text
Redesign the zaps.work homepage for mobile.

Priority order:
1. logo and compact nav
2. headline
3. search input
4. available tool card
5. coming-soon tool cards
6. compact quote result preview

Mobile rules:
- no horizontal scrolling
- no tiny unreadable calculator preview
- keep buttons single-line
- disabled tools must still look disabled
- quote preview can be simplified

The page should feel like a tool you can use immediately.
```

## 10. Production UI Prompt

```text
Implement a production-ready homepage for zaps.work.

Use:
- semantic HTML
- responsive CSS
- light and dark themes
- accessible buttons and form labels
- disabled states for unavailable tools
- no external UI illustration assets

Design direction:
- practical quote tool
- clean workbench
- Vercel-like restraint
- fintech dashboard polish
- green accent only where meaningful

Do not build a generic AI SaaS landing page.
Do not add chat UI, prompt input, abstract blobs, or heavy gradients.
```

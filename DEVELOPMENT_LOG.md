# zaps.work 开发日志

## v0.5.0 — 三个可解释的报价新场景

发布日期：2026-07-14
版本状态：已发布

本版新增自由职业报价、CNC 加工成本和草坪服务报价三个工具。它们继续遵守同一个产品边界：根据经营者输入的成本、时间和目标毛利计算价格，不以不可解释的市场均价代替业务判断。

### 本版新增

- 自由职业报价：将范围工时、修改轮数、每轮修改时间、固定成本、加急溢价、最低收费、目标毛利与税额放进同一份客户报价。
- CNC 加工成本：面向人工输入的 3 轴铣削估算，区分材料毛坯与损耗、编程、设置、单件循环、检验、机时、人工、刀具、后处理和批量分摊；不替代 CAM 或工艺规划。
- 草坪服务报价：根据可修剪面积、草况、团队产能、人工、设备与燃料、交通、附加服务、间接成本、最低收费和毛利生成报价。
- 三个工具均提供多语言路由、货币换算、输入归一化、PDF 打印／保存、CSV 导出、报价预览和移动端报价条。
- 新增三篇英文操作指南，并将工具和指南分别加入工具 Sitemap、指南 Sitemap 与 `llms.txt`。

### 下一步

继续完善油漆、CNC 与新服务场景的本地化操作指南和长尾页面；基于真实搜索与使用数据决定下一批垂直报价工具。

## v0.4.0 — 四个报价工具可信度修复与房屋油漆报价

发布日期：2026-07-14
版本状态：已发布

本版针对真实用户验收中发现的信任问题进行统一加固，覆盖 3D 打印、压力清洗、雷射切割和住宅清洁：货币不再只更换符号，物理单位不再只更换标签，无效数值不会让页面崩溃，展示为价格依据的字段也会真正进入计算。同时新增房屋油漆报价计算器，让服务业报价场景继续沿着透明、可解释的成本模型扩展。

### 本版改进

- 按所选货币缩放人工、运营、耗材、交通、最低收费和附加项起始价格，并使用正确的小数位格式。
- 非英语地区默认使用平方米；平方英尺与平方米切换时同时换算面积和生产率。
- 房屋类型、卧室、卫生间、清洁类型、难度与服务频率共同影响面积模式的基础人工时。
- 对负数、空值与非有限数字进行安全归一化，避免计算校验错误导致整页崩溃。
- 手机端增加始终可见的实时报价条，可直接跳转到完整结果与导出操作。
- 清空示例公司名、补齐报价详情开关名称，并明确附加项价格为“0 时自动计算”。
- 同步更新 8 种语言的方法说明与居家清洁定价指南公式。
- 3D 打印、压力清洗与雷射切割同步采用货币换算、货币标准小数位和空白公司名称。
- 压力清洗的表面状况、通行难度和最低收费正式进入报价模型，人工与运营成本默认开启。
- 雷射切割的材料种类和厚度正式进入材料估算；公英制切换会换算面积、面积单价、厚度与切割长度。
- 四个工具统一使用“打印／保存 PDF”文案，并提供移动端实时报价条。
- 补充异常输入、货币换算、单位换算和有效字段回归测试，并同步更新雷射切割操作指南。
- 新增房屋油漆报价计算器：用可上漆面积、涂层数、容器覆盖率、损耗、人工产能、表面状态、准备、间接成本、最低工程费、目标毛利和税额建立报价。
- 房屋油漆工具支持 8 种语言、平方英尺／平方米与货币真实换算、打印／保存 PDF、CSV 导出和移动端实时报价条。
- 新增英文《How to Price House Painting Jobs》操作指南，并通过工具页、指南 Sitemap 与 `llms.txt` 建立索引入口；其他语言版本和室内／室外长尾页进入下一阶段。

### SEO 发布检查

本版新增房屋油漆工具与英文操作指南：工具 Sitemap 已加入所有语言的工具路径；指南 Sitemap 仅加入已存在的英文主指南，不生成不存在的多语言 URL；`llms.txt` 已更新新工具、指南、货币换算、物理单位换算和打印／保存 PDF 能力。

## v0.3.0 — 从清洁人工时到客户固定报价

发布日期：2026-07-13
版本状态：已发布

本版增加住宅清洁报价生成器。用户可以按面积和生产率估算总清洁人工时，也可以由有经验的经营者直接输入人工时，再加入人工、营运分摊、耗材、交通、附加服务、最低收费、周期折扣与目标毛利。

### 本版新增

- 上线独立屋、公寓和集合住宅的清洁报价模型。
- 支持标准、深度、搬入／搬出清洁，以及单次和周期服务。
- 清楚区分总 cleaner-hours 与团队现场时长。
- 支持烤箱、冰箱、室内窗户、橱柜、洗衣和宠物毛发等附加服务。
- 支持目标毛利、最低服务费、首次清洁附加费、周期折扣与税额。
- 支持客户报价预览、PDF、CSV 与复制摘要。
- 加入隐私安全的清洁工具匿名导出事件，不采集客户姓名、地址或联系方式。
- 提供 8 种语言工具页面和《How to Price House Cleaning Jobs》操作指南。
- 同步更新工具 Sitemap、指南 Sitemap、`llms.txt`、结构化数据和工具／指南双向内链。

### 同期内容更新

- 新增 8 种语言的《How to Price Laser Cutting Jobs》指南。
- 将雷射切割指南加入指南首页、指南 Sitemap、`llms.txt` 和工具页内链。

### 下一步

生产发布并验证清洁报价的默认工时、附加服务和最低收费，再进入房屋油漆报价计算器。

## v0.1.0 — 从成本计算到专业报价

发布日期：2026-07-11
版本状态：已发布

这是 zaps.work 的第一个公开版本。

我们从一个很具体的问题开始：小型经营者和创客知道材料用了多少、机器运行了多久，却不一定能快速算清一笔订单的真实成本，更难判断报价能否达到目标利润。

首个上线工具是 **3D 打印成本计算器**。它把材料、打印时间、设备费率、人工、损耗和失败风险放进同一个透明的计算过程，再根据目标毛利生成建议报价。用户可以添加多个项目，并把结果导出为 PDF 报价单或 CSV 文件。

### 本版新增

- 上线 3D 打印成本计算器，支持多项目成本与报价计算。
- 支持材料、设备、人工、损耗、失败风险和其他成本的独立配置。
- 根据目标毛利计算建议报价，并展示成本构成。
- 支持 PDF 报价单与 CSV 数据导出。
- 提供英语、繁体中文、德语、日语、西班牙语、法语、巴西葡萄牙语和韩语版本。
- 加入可搜索的语言与货币选择器，并根据语言地区提供合理的货币默认值。
- 完成响应式布局，让手机用户可以先看到计算表单和关键结果。
- 上线隐私说明、站点分析、健康检查、站点地图、`robots.txt` 与 `llms.txt`。
- 开放源代码与 GitHub 反馈入口，计算逻辑和数据边界可以被公开检查。

### 我们特别在意的事

**计算必须透明。** zaps.work 不用一个看不见的“市场均价”替用户做决定。每一项成本、费率和利润目标都可以查看和调整。

**客户资料默认留在浏览器。** 姓名、邮箱、电话、地址、公司信息、报价编号和备注等敏感内容不会进入产品分析数据库。我们只记录经过白名单限制的匿名使用数据，用来判断工具是否真正有帮助。

**工具应该先能用，再讲故事。** 在手机和桌面端，计算器都尽量出现在页面最前面。我们也修复了数字输入、步进按钮、移动端排列和长列表滚动等首版体验问题。

### 幕后建设

- 使用 Next.js、React 与 TypeScript 建立多语言工具平台。
- 将计算逻辑拆分为可测试的纯函数，并加入单元测试与发布质量检查。
- 使用 Vercel 承载预览与生产环境，使用 Supabase 保存隐私安全的匿名分析事件。
- 建立 canonical、`hreflang`、多语言路由和搜索引擎索引基础。

### 下一步

接下来我们会继续打磨 3D 打印从“成本输入”到“客户报价”的完整流程，并验证更多真实工作场景。正在研究和开发的方向包括压力清洗、清洁服务、激光切割等，但它们会在计算模型、报价内容和实际用户验证完成后再公开发布。

如果你正在用 zaps.work 处理真实报价，欢迎通过 [GitHub](https://github.com/wudanglaodao/zaps-work-quote) 提交反馈。

## v0.2.0 — 激光切割报价垂直切片

发布日期：2026-07-13
版本状态：已发布

第二个制造场景围绕激光切割报价展开。它沿用 zaps.work 的通用报价文档和隐私边界，同时把激光切割真正需要的面积、材料厚度、切割长度、穿孔数量、机器时间、准备和后处理成本放进一个可调整的模型。

### 本版新增

- 上线激光切割成本计算器与报价单页面。
- 支持金属、亚克力、木板等材料分类，以及平方英尺和平方米单位。
- 支持材料损耗、切割时间、穿孔数量、机器费率、准备时间、搬运人工和后处理成本。
- 可选成本区默认关闭，减少首次填写干扰；打开后才展示额外成本字段。
- 沿用 PDF、CSV 和复制摘要导出，并把激光切割的匿名汇总事件纳入分析白名单。
- 加入所有已支持语言的本地化页面、canonical、`hreflang`、FAQ 结构化数据和工具站点地图入口。

### 下一步

下一步不扩展发票、收据或付款管理工作流，继续增加透明、可解释的垂直报价单与成本计算器场景。当前顺序为：

1. 清洁服务报价生成器。
2. 房屋油漆报价计算器。
3. 手工参数版 CNC 加工成本计算器。
4. 根据搜索和使用数据，从园艺／草坪服务、汽车美容等候选场景中选择下一项。

每个新工具都必须同步完成页面级 SEO、8 种语言页面、canonical 与 `hreflang`、结构化数据、工具 Sitemap、指南 Sitemap、`llms.txt` 和至少 1 篇对应的操作或定价指南。工具、指南和相关场景之间需要建立内链；这些内容属于同一次发布，不作为上线后的补充任务。

## 后续版本记录模板

> 发布新版本时复制下面的结构，将新版本放在本文顶部，并删除不适用的小节。

```markdown
## vX.Y.Z — 一句话说明本版价值

发布日期：YYYY-MM-DD
版本状态：已发布

用两三句话说明这个版本解决了什么问题，以及用户为什么会在意。

### 本版新增

- 用户可以感知的新功能或变化。

### 体验改进

- 操作、性能、移动端、可访问性或内容方面的提升。

### 问题修复

- 对用户有影响的重要修复，不需要罗列所有内部提交。

### 幕后建设

- 值得公开说明的架构、测试、隐私、安全或发布能力。

### 下一步

简要说明已经确认的方向，避免承诺未经验证的发布日期。
```

## 记录约定

- 使用语义化版本号：重大不兼容变化为主版本，新功能为次版本，修复为补丁版本。
- 只记录已经进入生产环境的内容；开发中的功能放在“下一步”，不要写成已经发布。
- 从用户价值开始写，内部重构只有在影响可靠性、速度、隐私或后续能力时才记录。
- 发布日期以生产环境上线日期为准。
- 每次上线后补充日志链接、最终版本号和必要截图，即可整理为博客文章。
